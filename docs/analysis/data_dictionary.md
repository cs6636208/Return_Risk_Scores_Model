# 📖 พจนานุกรมข้อมูล (Data Dictionary): ระบบทำนายความเสี่ยงการคืนสินค้า

เอกสารฉบับนี้รวบรวมคำอธิบายรายละเอียดของข้อมูล (Columns) ที่ใช้ในโครงการทำนายความเสี่ยงการคืนสินค้า (Return Risk Prediction) เพื่อให้ทีมธุรกิจและทีมพัฒนาเข้าใจตรงกัน

---

## 1. ข้อมูลธุรกรรมหลัก (Core Transactional Data - Orders)
ข้อมูลที่เกี่ยวข้องกับการสั่งซื้อแต่ละรายการ

| ชื่อคอลัมน์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `order_id` | String | รหัสอ้างอิงการสั่งซื้อที่ไม่ซ้ำกัน (Primary Key) |
| `customer_id` | String | รหัสสมาชิกลูกค้า |
| `product_id` | String | รหัสสินค้า |
| `courier_id` | String | รหัสบริษัทขนส่งที่ให้บริการ |
| `promo_id` | String | รหัสโปรโมชั่นที่ใช้ในคำสั่งซื้อนี้ |
| `order_date` | DateTime | วันและเวลาที่ลูกค้ากดสั่งซื้อ |
| `expected_delivery_date` | DateTime | วันและเวลาที่คาดว่าจะส่งมอบสินค้า |
| `delivery_date` | DateTime | วันและเวลาที่สินค้าถูกส่งถึงมือลูกค้า |
| `channel_type` | Category | ช่องทางการขาย (เช่น TV_Show, Mobile_App, TikTok, Shopee) |
| `payment_method` | Category | วิธีการชำระเงิน (เช่น COD, บัตรเครดิต, โอนเงินผ่านธนาคาร) |
| `quantity` | Integer | จำนวนสินค้าที่สั่งซื้อในรายการนั้น |
| `unit_price` | Float | ราคาสินค้าต่อหน่วย ณ เวลาที่สั่งซื้อ |
| `tier_discount_pct` | Float | เปอร์เซ็นต์ส่วนลดจากระดับสมาชิก (เช่น 0.05 หมายถึง 5%) |
| `campaign_discount_pct` | Float | เปอร์เซ็นต์ส่วนลดจากโปรโมชั่นสินค้า (เช่น 0.05 หมายถึง 5%) |
| `discount_applied_amount` | Float | มูลค่าส่วนลดรวมที่เป็นจำนวนเงิน (บาท) |
| `total_amount` | Float | ยอดชำระเงินสุทธิหลังจากหักส่วนลดแล้ว |
| `delivery_days` | Integer | จำนวนวันที่ใช้ในการขนส่งจริง |
| `order_hour` | Integer | ช่วงเวลาที่สั่งซื้อ (0-23) |

## 2. โปรไฟล์และพฤติกรรมลูกค้า (Customer Profile & Behavior)
ข้อมูลที่ใช้ระบุลักษณะและพฤติกรรมการซื้อย้อนหลังของลูกค้า

| ชื่อคอลัมน์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `customer_name` | String | ชื่อ-นามสกุล ของลูกค้า |
| `customer_phone` | String | เบอร์โทรศัพท์ (สำหรับใช้ในกรณี Call Center ติดต่อ) |
| `membership_tier` | Category | ระดับสมาชิก (Gold, Silver, Bronze, Platinum) |
| `preferred_channel` | Category | ช่องทางการซื้อประจำที่ลูกค้าถนัด |
| `province` | Category | จังหวัดที่จัดส่งสินค้า |
| `registration_date` | DateTime | วันที่ลูกค้าลงทะเบียนเป็นสมาชิกครั้งแรก |
| `customer_age_days` | Integer | จำนวนวันที่เป็นสมาชิก (นับจากวันสมัครถึงวันสั่งซื้อ) |
| `days_since_last_order`| Integer | ระยะห่างจากการสั่งซื้อครั้งล่าสุด (ถ้าเป็นครั้งแรกจะเป็น -1) |
| `delivery_time_expected_days` | Integer | ระยะเวลาขนส่งที่คาดการณ์ (เช่น 1-3 วัน) |
| `hist_order_count` | Integer | จำนวนการสั่งซื้อสะสมทั้งหมดก่อนหน้าธุรกรรมนี้ |
| `hist_return_rate` | Float | อัตราการคืนสินค้าสะสม (0.0 - 1.0) ก่อนหน้าธุรกรรมนี้ |
| `is_repurchased_item` | Boolean | ระบุว่าสินค้าชิ้นนี้ลูกค้าเคยซื้อไปแล้วหรือไม่ (การซื้อซ้ำ) |

## 3. ข้อมูลสินค้าและซัพพลายเออร์ (Product & Supplier Information)
รายละเอียดของสินค้าและที่มาของสินค้า

| ชื่อคอลัมน์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `product_name` | String | ชื่อเรียกของสินค้า |
| `category` | Category | หมวดหมู่สินค้า (เช่น แฟชั่น, เครื่องใช้ไฟฟ้า, สุขภาพ) |
| `brand` | String | แบรนด์สินค้า |
| `is_fragile` | Boolean | ระบุว่าเป็นสินค้าแตกหักง่ายหรือไม่ (ต้องดูแลเป็นพิเศษ) |
| `product_rating` | Float | คะแนนรีวิวเฉลี่ยของสินค้าชิ้นนั้น |
| `supplier_name` | String | ชื่อบริษัทผู้ผลิตหรือจัดจำหน่ายสินค้า |
| `courier_name` | String | ชื่อบริษัทขนส่ง |

## 4. ข้อมูลการคืนสินค้า (Return & Refund - ตัวแปรเป้าหมาย)
ข้อมูลที่ระบบใช้เป็นคำตอบ (Target) ในการฝึกสอน Model

| ชื่อคอลัมน์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `is_returned` | Boolean | **ตัวแปรเป้าหมาย**: 1 = คืนสินค้า, 0 = ไม่คืนสินค้า |
| `return_id` | String | รหัสการคืนสินค้า (จะเป็น NULL หากไม่มีการคืน) |
| `return_date` | DateTime | วันที่เริ่มดำเนินการคืนสินค้า |
| `return_reason` | Category | เหตุผลที่คืนสินค้า (เช่น สินค้าชำรุด, ไม่ตรงปก, เปลี่ยนใจ) |
| `item_condition` | Category | สภาพสินค้าตอนรับคืน (เช่น ยังไม่แกะซีล, กล่องพัง, สินค้ามีตำหนิ) |
| `return_status` | Category | สถานะการคืนสินค้า (เช่น ดำเนินการเสร็จสิ้น) |
| `refund_amount` | Float | จำนวนเงินที่คืนให้ลูกค้าทั้งหมด |

## 5. ข้อมูลความเสี่ยงจากการวิเคราะห์ (Model Risk Metadata)
ผลลัพธ์ที่ได้จาก Model ทำนายผล

| ชื่อคอลัมน์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `risk_score` | Float | คะแนนความน่าจะเป็นที่จะเกิดการคืนสินค้า (0.0 - 1.0) |
| `risk_tier` | Category | ระดับความเสี่ยง (Low, Medium, High) |
| `shap_values` | JSON String | ปัจจัยหลักที่ทำให้ Model ตัดสินใจให้คะแนนความเสี่ยงนั้น |

## 6. ข้อมูลโปรโมชั่น (Promotions)
ข้อมูลแคมเปญส่งเสริมการขายเพื่อติดตามที่มาของส่วนลด

| ชื่อคอลัมน์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `promo_id` | String | รหัสโปรโมชั่น (Primary Key) |
| `promo_name` | String | ชื่อแคมเปญ (เช่น 11.11, Silver Member Perk) |
| `promo_type` | Category | ประเภทของโปรโมชั่น (เช่น Tier, Campaign, Flash Sale) |
| `discount_rate` | Float | อัตราส่วนลดสำหรับแคมเปญนี้ |
| `start_date` | DateTime | วันที่เริ่มโปรโมชั่น |
| `end_date` | DateTime | วันที่สิ้นสุดโปรโมชั่น |

## หมายเหตุเรื่องการคำนวณส่วนลด (Discount Logic)
- **สูตรการคำนวณแบบส่วนลดทบซ้อน (Stackable Discounts)**: 
    - `total_discount_pct` = `tier_discount_pct` + `campaign_discount_pct`
    - `discount_applied_amount` = `(unit_price * quantity) * total_discount_pct`
    - `total_amount` = `(unit_price * quantity) - discount_applied_amount`
- **กรณีการได้รับส่วนลด**:
    1. **ตามระดับสมาชิก (Membership Tier)**: หากลูกค้าอยู่ในระดับ Bronze ขึ้นไป อาจได้รับส่วนลดคงที่ 5% ในทุกหมวดหมู่สินค้า / หากลูกค้าอยู่ในระดับ Silver ขึ้นไป อาจได้รับส่วนลดคงที่ 10% ในทุกหมวดหมู่สินค้า / หากลูกค้าอยู่ในระดับ Gold ขึ้นไป อาจได้รับส่วนลดคงที่ 15% ในทุกหมวดหมู่สินค้า / หากลูกค้าอยู่ในระดับ Platinum ขึ้นไป อาจได้รับส่วนลดคงที่ 20% ในทุกหมวดหมู่สินค้า (บันทึกใน `tier_discount_pct`)
    2. **ตามแคมเปญสินค้า (Product Promotion)**: ส่วนลดเฉพาะตัวสินค้าตามช่วงเวลาส่งเสริมการขาย (บันทึกใน `campaign_discount_pct`)
