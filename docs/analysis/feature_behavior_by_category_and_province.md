# 📊 Feature Behavior Analysis by Category & Province

รายงานนี้วิเคราะห์พฤติกรรมและการกระจายตัวของ Features แต่ละตัวเมื่อเปลี่ยนตาม **Category** ใน 8 จังหวัดเป้าหมาย
เพื่อให้เข้าใจชัดเจนว่าเมื่อระบบเปลี่ยนหมวดหมู่สินค้า ค่าของ Features ที่เราสร้างขึ้นมาวิเคราะห์ความเสี่ยงการคืนของ (Return Risk) จะมีลักษณะเปลี่ยนแปลงไปอย่างไร

## 📌 1. สรุปพฤติกรรมหลัก (Key Logic Rules)
- **Features เฉพาะของแฟชั่น (`is_fashion_tv`, `is_bracketing`, `is_impulse_buy`, `is_perfect_storm`):**
  - จะมีค่าเฉลี่ยมากกว่า 0 เฉพาะเมื่อ Category = `Fashion` เท่านั้น
  - เมื่อเปลี่ยนเป็น `Supplement` หรือ `Home_Appliance` ค่าของ Features เหล่านี้จะกลายเป็น **0.00** เสมอ เพราะเงื่อนไขผูกติดกับหมวดหมู่แฟชั่น
- **Features เฉพาะของพื้นที่การจัดส่ง (`is_high_friction`, `is_long_distance_cod`):**
  - `is_long_distance_cod` จะเป็น 1 เฉพาะใน 3 จังหวัดที่เป็นหัวเมืองท่องเที่ยวไกล (`Chiang Mai`, `Phuket`, `Songkhla`) ที่สั่งแบบ COD (เก็บเงินปลายทาง) โดยไม่ขึ้นกับ Category
  - `is_high_friction` (จัดส่งช้า + อยู่ในจังหวัดเสี่ยง ได้แก่ Chiang Mai, Phuket, Songkhla, Remote_Area) จะเปิดทำงานในพื้นที่เหล่านี้เมื่อมีการส่งของเลท (Delivery Gap > 0)
- **Features ทั่วไป (`is_cod`, `is_low_commitment`, `delivery_gap`):**
  - เปลี่ยนแปลงไปตามพฤติกรรมการสั่งซื้อและการขนส่งจริงในแต่ละ Category x Province
